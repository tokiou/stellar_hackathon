Features

Everything you need, nothing you don't

Compass combines conversational AI with multi-layer security so every on-chain action is verified before it's signed.

Conversational Wallet

Just tell the wallet what you want. "Swap 200 USDC to SOL" or "Send 0.5 SOL to Alice" — the agent handles the rest, showing a rich proposal card before any signing.

Token Risk Scoring

Every swap is analyzed: liquidity depth, holder concentration, mint/freeze authority, Token-2022 extensions, blocklists, and price impact — all scored before you sign.

Conditional Orders

"Buy SOL when it drops below $130." The keeper monitors the market and executes automatically through an on-chain conditional escrow contract — no babysitting required.

On-chain Enforcement

The AgentActionGuard program verifies every critical action on-chain — hashes, signatures, expiry, and anti-replay — so guardrails can't be bypassed at the protocol level.

Self-Custodial

Your keys, your crypto. The backend prepares unsigned transactions; Phantom signs. The backend never holds your private key or signed transactions.

Full Audit Trail

Every action is logged with the canonical quote hash, risk report hash, oracle signature, provider sources, and on-chain tx signature — fully auditable after the fact.

How it works

Zero trust, every trade

Every operation goes through a multi-layer evaluation pipeline before a single satoshi moves.

1

You express intent

Type a natural language request — swap, transfer, or set a conditional order. The AI agent parses your intent and kicks off evaluation.

2

Off-chain risk engine runs

The backend queries Jupiter, Birdeye, and Helius in parallel. It calculates a risk score (0–100), checks blocklists, token extensions, and liquidity depth. A signed TokenRiskReport is generated with a canonical hash.

3

You review the proposal

A rich card shows the quote, risk score, warnings, and expiry. REJECT stops here. WARN requires explicit acknowledgment. APPROVE is ready to sign.

4

On-chain guard enforces

The AgentActionGuard program verifies hashes, the oracle signature, expiry, and anti-replay before executing the CPI. If anything changed since preview, the transaction is rejected on-chain.

Security

Guardrails at every layer

Off-chain risk signals feed an on-chain program that enforces every approval cryptographically.

AgentActionGuard Program

The Solana program at the core of every guarded action. It validates the oracle signature, checks PDAs for active approvals and wallet attestations, enforces expiry, and prevents replay attacks — all on-chain.

ActionApproval PDA WalletSafetyAttestation PDA guarded_transfer anti-replay nonce

ConditionalEscrowBuy Program

Locks funds in a devnet escrow contract. The keeper monitors prices and triggers execution only when your price condition is met — fully auditable and non-custodial.

devnet: FDwv…Ut7u escrow account price oracle

Oracle-signed Risk Reports

The backend signs every TokenRiskReport with an authorized oracle key. The on-chain program verifies the signature and the canonical hash so the frontend can't slip in a different quote.

riskReportHash quoteHash oracleSignature policyVersion

Hard Reject Rules

Certain conditions always block the action, no matter the context: blocklisted tokens, scam-flagged mints, active freeze authority on non-allowlisted tokens, dangerous Token-2022 extensions, or expired reports.

blocklist check freeze authority Token-2022 ext freshness window
