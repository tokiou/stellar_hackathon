Current backend modules

The active tree is a backend library/core. It is cleanly separated from the old chat product, but not yet exposed through an MCP server or API route.

Gateway primitives

back/services/executionGateway.ts

  • Classifies tool calls by risk class.
  • Creates canonical action candidates.
  • Builds redacted audit envelopes.

Policy engine

back/services/policy/*

  • Loads and validates defaultPolicy.yaml.
  • Evaluates allow / deny / approval outcomes.
  • Fails closed for signing or unknown mutating tools.

Transfer guard

transferGateway.ts + contracts

  • Derives policy context and fingerprints.
  • Builds approval metadata.
  • Verifies metadata before execution.

Wallet safety

walletSafetyValidation.ts

  • Recipient and amount risk checks.
  • Deterministic action hashes.
  • Attestation/on-chain evidence hooks.

On-chain approval

onchainApproval.ts

  • Derives and checks approval PDAs.
  • Validates wallet safety attestation PDAs.
  • Verifies guarded-transfer readiness.

Price context + audit

priceQuote.ts, transferAuditLog.ts

  • Devnet USDC/SOL quote via Orca provider.
  • Optional fallback price for tests/dev.
  • Bounded in-memory audit log.

Known debts

These are not regressions; they are the next blockers to turn the backend core into a product surface.

blocker

No MCP/tool boundary

No active API/MCP entrypoint consumes transferGateway. Wave 4 now targets a first-party MCP server/tool boundary.

runtime debt

Audit is in-memory

transferAuditLog.ts is bounded and useful for tests/dev, but not durable enough for real audit requirements.

test debt

On-chain integration tests

Backend unit tests pass, but Anchor flows still need real integration coverage or a deterministic local validator suite.

scope decision

Conditional escrow

The program remains active under back/solana/, while its old app/keeper wiring lives in legacy/.

network debt

Devnet quote assumptions

Orca pool/mints are devnet-specific; fallback price must not silently drive mainnet policy decisions.

docs debt

Task metadata stale

Wave 2 task status and Wave 3 caller references need post-merge/post-isolation updates.
