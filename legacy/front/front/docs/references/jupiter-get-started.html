Referencia Jupiter — archivada para frontend

Esta copia local de documentación de Jupiter no debe usarse como guía de implementación del frontend.

Según front/docs/frontend-spec.md, el frontend no llama directamente a Jupiter. Quotes, routing, fallbacks y cualquier API key pertenecen al backend/agent.

Uso correcto

  • Frontend: consumir solo endpoints propios como /api/chat o /api/wallet/*.
  • Backend/agent: si necesita Jupiter, documentarlo e implementarlo fuera de front/src.
